User Request:
Let's build a new app named Sezzle Calculator Test.

Objective
Build a full-stack calculator application with a React frontend and a backend microservice. The frontend should consume the backend API to perform basic and advanced arithmetic operations. It must implement a clean design, maintainable code, and testable architecture. This will have just one single page, with the design of a common calculator (buttons for numbers from 0 to 9 in order, operators, clear screen, dot, and the screen with the result where the operations will be shown.

Functional
-Operations:
 Addition, Subtraction, Multiplication, Division, Exponentiation, Square Root, Percentage.
-Frontend (React):
 Intuitive UI for entering input and displaying results.
 Input validation and error handling.
 Responsive design (basic mobile support).
-Backend (REST API):
 Expose endpoints for calculator operations.
 Validate input and handle edge cases (division by zero, invalid data).
 Return results in JSON format.
-Non-Functional
 Clean, readable, and idiomatic code (frontend and backend).
 Unit tests covering key functionality for both layers.
 Documentation: setup instructions, API usage, and design rationale.
 Dockerfile for full-stack deployment.

Constraints
Frontend: React (TypeScript)
Backend: Go

Extra Deliverables
Generate a README file with setup instructions (How to run frontend and backend), API examples, and design decisions.
Unit tests and coverage report
Dockerfile to run frontend + backend together
Save the propmts for this project in a txt file.
